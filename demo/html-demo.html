HTML 规范

1. DOCTYPE 位置

<DOCTYPE html > 必须在文档的最开始，避免引起文档的怪异渲染模式。

2. 移动端的 meta 值

<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
<meta content="email=no" name="format-detection" />
<meta name="format-detection" content="telephone=no" />

3. PC 端 IE edge

<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1"> 。

4. 标签闭合

基础要求。

5. 不使用 inline 样式。

禁止在页面的head通过 <style> 定义样式，和在 dom 中通过使用 style 的方式定义 inline 样式。

6. 4个空格缩进

使用4个空格代替制表符缩进。

7. 注释规则

模块通过 <-- 模块名字 -- > 开始，<-- /模块名字 -- > 结束，状态切换注释在DOM切换处注释标注。

8. 媒体查询

Tv

Tv端测试出的device-width有三种：device-width:1920px device-width:1280px device-width:960px; UI输出的页面尺寸通常为1920*1080 在开发中是以device-width:960px开发的，需增加如下媒体查询条件适配另外两种情况： 

	@media screen and (min-width:1280px) and (max-width:1919px){
		html,body{font-size:83.3%;}
	}
	@media screen and (min-width:1920px){
		html,body{font-size:125%;}
	}

Mobile

通常移动端测试涵盖的设备范围是320-768px。可以理解为iPhone5--iPad的设备之间，安卓设备通常的最小webview是360px 

	@media screen and (max-width:768px){
		......
	}

9. 性能优化

  • 减少请求数是优化不变的宗旨。请衡量开发、维护成本
  • 图片压缩。jpg（大背景、照片）、png（需要透明、质量高）、gif。图片较多时请考虑懒加载 线上压缩
  • 合理的使用标签、结构
  • 代码的复用及扩展性也是不能忽略的。好的复用及扩展可减少代码量，增强统一性
